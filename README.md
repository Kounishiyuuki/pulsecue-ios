# PulseCue

**ワークアウトと栄養を、端末内だけで記録・可視化するローカルファーストの iOS アプリ**

SwiftUI + SwiftData 製 / 外部ライブラリ・バックエンド・アカウント登録なし / iOS 17+

</div>

> 進行中の個人 iOS ポートフォリオプロジェクトです。

---

## PulseCue とは

PulseCue は、**トレーニングの実行**と**日々の栄養・体調**を 1 つのアプリで管理するための
iOS アプリです。

- **誰のためか** — ジムでのワークアウトを習慣にしたい人、食事のカロリー / タンパク質を
  ゆるく記録したい人。
- **何を解決するか** — 「次に何をやるか」を考える負荷と、食事記録の入力の手間。ランナー画面が
  種目と休憩のテンポを刻み、バーコード / 栄養表示 OCR / 写真が食事入力を補助します。
- **なぜローカルファーストか** — 健康・食事・写真は機微なデータです。PulseCue はそれらを
  端末内の SwiftData にのみ保存し、ネットワーク送信・アカウント登録を行いません。
- **なぜ「レビューしてから保存」か** — バーコードや OCR、将来の AI 推定は誤ることがあります。
  推定結果は必ず「候補」として確認画面に出し、ユーザーが確定したものだけを記録します。

## 主な機能

### トレーニングの実行支援
- ルーティン / ステップの作成・編集・複製・並び替え・ピン留め・検索
- ランナー（実行画面）— `Now / Rest / Next` の固定 UI と Complete / Skip / +10 / Back
- 休憩タイマーは締切時刻ベース。バックグラウンド復帰やアプリ再起動後も継続
- 履歴 — 過去セッションとステップ結果の振り返り

### 毎日の健康ダッシュボード（今日）
- 摂取カロリー / 運動消費カロリー / 睡眠 / 体重を 1 画面に集約
- カロリー収支（摂取 − 消費）を自動計算
- 健康目標との差分表示（摂取・睡眠・運動消費・収支・体重）
- 直近 7 日の平均と体重トレンドを健康サマリーで表示

### 食事ログ
- 朝食 / 昼食 / 夕食 / 間食ごとの手動入力
- 最近の食事・よく使う食事テンプレートからワンタップで今日に追加
- カロリー / タンパク質の合計（確定済みの食事のみ集計）

### 入力アシスト
- **バーコード商品照会** — 商品バーコードから栄養情報を取得し、確認画面を経て保存
- **栄養表示ラベル OCR** — パッケージの栄養成分表示を端末内で読み取り、確認画面を経て保存
- **食事写真キャプチャ（プロトタイプ）** — 写真の選択 / 撮影とプレビュー
- **写真推定の候補フロー（モック）** — 写真 → 候補 → 確認画面 → 保存 のフローをモック推定で先行実装

### 健康目標のカスタマイズ
- デフォルト目標の設定
- 曜日ごとの上書き
- 日付指定の上書き（旅行・外食など特定日だけ目標を変える）

### 安全性・プライバシー
- ローカルファースト（端末内保存のみ）
- 確定済みの食事だけを合計に加算
- 写真の暗黙アップロードなし
- AI プロバイダの API キーを iOS アプリに埋め込まない方針

## アプリの設計

シンプルな言葉で言うと、**「推定は候補にすぎず、ユーザーが確定したものだけが記録になる」**
を構造で保証しています。

- **`MealEntry` の status / source モデル** — 各食事は状態（`pending` 確認待ち / `confirmed`
  確定済み）と発生源（`manual` / `ai` / `barcode` / `ocr`）を持ちます。enum を raw 文字列で
  保持するため、源の種類が増えてもスキーマ移行は不要です。
- **`NutritionLedger`** — `DayLog` の摂取カロリーを「確定済み `MealEntry` の合計」で再計算する
  唯一の経路。確認待ちの候補は加算されません（カロリーの単一の真実）。
- **`ProteinTotals`** — タンパク質合計も確定済みのみを読み取り時に集計します。
- **`PhotoFoodEstimating` 抽象化** — 写真推定プロバイダのプロトコル境界。実装は決定論的・
  オフラインのモックのみで、将来の実プロバイダを差し込めるようにしてあります。
- **候補 → レビュー → 確定 → 保存** — バーコード / OCR / 写真のいずれも、候補は確認画面で
  ユーザーが明示的に確定したときだけ `MealEntry` になり、その時点で `NutritionLedger` 同期まで
  行います。確認前に DayLog や合計へ書き込む経路は存在しません。

## 技術ハイライト

- **SwiftUI** によるコンポジション中心の UI 構築
- **SwiftData** によるローカル永続化（iOS 17+）
- **Apple Vision** による端末内 OCR（栄養表示ラベルの文字認識）
- **AVFoundation** によるバーコード読み取り（カメラセッション）
- **PhotosPicker / UIImagePickerController** による写真選択・カメラ撮影
- **UserDefaults** ベースの軽量ストア（テンプレート・並び順・健康目標・ランナー復帰状態）
- **Swift Testing / XCTest** による単体テスト（350 件超）と起動スモークテスト
- ローカルファーストの安全設計と、実 AI の前に用意したプロバイダ抽象化

---

## 開発プロセスと品質

- 小さな **PR 単位のインクリメンタル開発**（1 機能 = 1 PR）
- 各 PR で `xcodebuild` の **ビルド / テスト検証**を実施
- バーコード / OCR / 写真の保存フローに「**確認前は何も保存しない**」テストを用意
- AI は**モックファースト** — フローと安全境界を先に固めてから実装に進む方針
- **安全第一のクレデンシャル戦略** — クライアントにプロバイダの秘密情報を置かない（設計を文書化）

## ビルドと実行

1. リポジトリのルートで `Pulse Cue.xcodeproj` を Xcode で開く
2. Scheme を `Pulse Cue` に設定する
3. iOS 17 以降の iPhone シミュレータを選び、`Run`（⌘R）

初回起動時にサンプルルーティンが自動投入されます。バーコード読み取りとカメラ撮影は実機で
確認でき、写真ライブラリからの選択はシミュレータでも動作します。設計・運用メモは `Docs/` に
あります。
