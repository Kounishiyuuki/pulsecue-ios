### 週間サマリーの目標差分（PR #38）

> 前提: 設定 → 健康目標 で デフォルト目標、必要なら 曜日上書き / 日付上書き を設定済み。週間平均は 3 日以上の入力が必要。

- [ ] 健康サマリーの「過去 7 日の平均（目安）」セクションで、目標が設定されているメトリックに「目標 X kcal/日」+ 差分（例: 「あと 350 kcal」）の小さい行が表示される
- [ ] 睡眠の差分行は「目標 7時間30分」+ 「目標まで あと 1時間10分」のように分/時間表示される
- [ ] バランスの差分行は「目標 -500 kcal/日」+ 「+200 kcal」のように負の目標も正しく表示される
- [ ] 目標が未設定のメトリックは差分行が出ず、PR #38 以前と同じ「平均値のみ」表示になる
- [ ] 入力が 3 日未満のメトリックは平均値そのものが「—」になり、目標差分も表示されない（既存ルール温存）
- [ ] 曜日上書きを設定 → 健康サマリーの目標平均がその曜日を含めて再計算される
- [ ] 日付指定の上書きを設定 → 該当日が 7 日範囲内なら目標平均に反映される
- [ ] 設定をクリア → 健康サマリーの差分行が消え、PR #38 以前の表示に戻る
- [ ] 今日タブのカード差分表示（PR #36）と 健康サマリーの週間差分は視覚的に分離され、Today は「今日」、HealthSummary は「7 日平均」を扱っている
- [ ] このセクションの操作中、ネットワーク通信は発生しない

## Nutrition: 食事カロリーの整合性（P0 reliability）

> Today の摂取値 / バランス と Nutrition タブの合計が**常に一致する**ことが要件。
> 確定 (`.confirmed`) のみが合計に加算される。
> 確認待ち (`.pending`, AI 推定) は合計に影響しない。

### 計算 / 整合性

- [ ] Nutrition タブで食事を追加 (例 朝食 400 kcal、昼食 800 kcal) → 合計 1200 kcal が即時反映される
- [ ] Today タブに戻ると 摂取 1200 kcal / バランス が同じ値で更新されている
- [ ] 1 件をスワイプで削除 → Nutrition の合計が新しい値 (800 kcal) になる + Today も同じ値に
- [ ] **最後の確定済み食事**を削除 → Today の摂取が空 (— 表示) になり、古い 800 kcal が残らない
- [ ] 食事を編集して kcal を変更 → Nutrition と Today の合計が新しい値に同時に追随する
- [ ] AI 推定 (確認待ち) を追加しても摂取合計は変わらない (確定するまでカロリーには加算されない)
- [ ] AI 推定を破棄しても確定済みの合計は変わらない
- [ ] AI 推定を確定 → 確定済みカロリーに正しく加算される
- [ ] アプリを再起動 → 同じ日の合計が保たれる
- [ ] 朝食 / 昼食 / 夕食 / 間食 をまたいで複数件追加 → 合計が正しく加算される
- [ ] 昨日の食事と今日の食事が混ざらない (ローカル日付でグルーピング)
- [ ] 23:59 と 00:01 の食事は別の日に振り分けられる (端末タイムゾーンで判定)
- [ ] 一連の操作中、外部通信は発生しない（AI ネットワーク呼出は無し）
