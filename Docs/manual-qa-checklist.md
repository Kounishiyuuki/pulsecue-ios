# 手動 QA チェックリスト（v2 / `main`）

Xcode で Scheme `Pulse Cue` を選択して以下を順に確認します。所要時間 10 分程度。

## 起動 / 全体

- [ ] ⌘R で起動 → 「今日 / ワークアウト / ランナー / 履歴 / 設定」の 5 タブが表示される
- [ ] 起動直後にホーム画面に戻る → AppIcon に PulseCue のパルス線アイコンが表示される（Dark / Tinted モードでも崩れない）
- [ ] アプリを kill → 再起動で SwiftData の状態が保持される（DayLog、ルーティン、履歴）

## Today タブ

- [ ] 6 カード（ワークアウト / 栄養 / 運動消費 / 睡眠 / 体重）+ バランス + 「週間サマリー →」リンク
- [ ] 未入力カードはオレンジ枠でハイライト、ボタンが「入力」表示
- [ ] 任意のカードを起動直後に即タップ → 空シートが出ず、入力フォームが表示される
- [ ] 値を保存 → カードが「編集」表示と新しい数値で即時更新される
- [ ] 摂取 / 消費を入力 → バランスカードが `intake − exercise` で更新される
- [ ] 「週間サマリー →」をタップ → `HealthSummaryView` の各セクションが表示される（データ不足時は `—`）

## Today: ジムからメニュー作成カード（P4）

- [ ] Today タブに「ジムからメニュー作成」カードが表示されている（ワークアウト開始ボタンと指標カード群の間）
- [ ] ジム未登録時: 「ジムを登録すると、設備に合わせたメニューを作成できます」+「ジムを登録する」CTA → タップで既存のジム検索/登録シートが開く
- [ ] ジムは登録済みだがアクティブなジムがない状態: 「今日使うジムを選択してください」+「ジムを選ぶ」→ `MyGymHomeView` が開ける
- [ ] アクティブなジム + マシン保存済み: ジム名 + 「N 台のマシンを使ってメニューを組みます」+「部位を選んで作成」→ タップで `TargetBodyPartSelectionView` に直接遷移
- [ ] アクティブなジムだがマシン未登録: 「このジムのマシン情報がまだありません」+「マシン情報を準備する」→ `MyGymHomeView` 経由でマシン選択にアクセスできる
- [ ] 公式URLが登録されているジムでマシン未登録のとき、補足文「公式URLからの自動取り込みは今後対応予定です。」が表示される
- [ ] 部位選択 → `GeneratedPlanPreviewView` → 「ルーティンとして保存」のフローが従来どおり動く
- [ ] Today タブの他カード（ワークアウト / 栄養 / 運動消費 / 睡眠 / 体重 / バランス / 週間サマリー）は変わらず動く
- [ ] Settings → マイジムからの既存フローは引き続き動く

## ランナー（最重要）

- [ ] 任意ルーティンで開始 → 「今 / 休憩 / 次」3 カード + 下部アクションバー
- [ ] Now カードに種目名 / セット / 回目標が表示される
- [ ] **Complete**：エクササイズ → 休憩へ。Complete ボタンが「休憩終了」に切替
- [ ] 休憩中の **+10** で残り秒が 10 秒延びる（数値がスムーズにロールアップ）
- [ ] 休憩中の **Complete（休憩終了）** で次のセットへ
- [ ] 休憩中の **Skip** でステップ全体をスキップして次のステップへ
- [ ] エクササイズ中の **Skip** で現在のステップをスキップ
- [ ] **Back** で 1 セット戻る、ステップ境界も跨げる
- [ ] アプリをバックグラウンド → 復帰 → 残り休憩秒が `deadlineDate` 基準で正しい
- [ ] アプリを kill → 再起動で in-progress セッションが復元される
- [ ] 最後のセットの最後の Complete → セッション完了 → 履歴に追加される
- [ ] 「セッション終了」ボタンで中断保存 → 履歴で「中断」として残る
- [ ] 通知許可済み + 休憩を実機で待機 → 終了通知が発火する（または iOS 設定で許可取消後に Settings を再表示するとトグルが自動 OFF）

## ジム候補検索: 現在地から（P3）

> シミュレータで動作確認するには Features → Location → Custom Location… で座標を設定する必要がある。

- [ ] アプリ起動時 / マイジムを開いた時 / 「ジムを追加」 シートを開いた時には位置情報の許可ダイアログが出ない
- [ ] 「ジムを追加」 シートの先頭に「現在地から近くのジムを探す」 行がある
- [ ] その行をタップ → 「現在地から探す」 画面に遷移する
- [ ] 初回タップで初めてシステムの位置情報許可ダイアログが出る（このタイミング以前には絶対に出ない）
- [ ] 「許可」 → 「位置情報を取得中…」 → 「近くのジムを検索中…」 → 候補リストが出る
- [ ] 候補をタップ → GymRegistration が名前 (+任意でURL) で埋まった状態で開く
- [ ] 「許可しない」 → 「位置情報の許可が必要です」 セクションが表示され、「設定アプリを開く」 ボタンが動作する
- [ ] 設定アプリで位置情報を ON にしてアプリに戻る → 画面は自動再検索しない。「もう一度近くを検索」 ボタンで再試行できる
- [ ] シミュレータで Location 未設定 → エラー表示 + 下の「見つからない場合は手動で入力する」 で従来の手動登録に進める
- [ ] 機内モード ON でも同様にエラー + 手動フォールバックが動く
- [ ] 「もう一度近くを検索」 ボタンで再検索できる
- [ ] PR #21 のテキスト検索 (「ジムを検索」 のフィールド) が引き続き動作する
- [ ] PR #20 の手動登録フロー (「見つからない…」 経由) が引き続き動作する
- [ ] 一連の操作中、Apple マップ以外への通信は発生しない（コンソールに PulseCue の Worker URL 等への通信は出ない）

## 既知の制限（v2 時点）

- AppIcon はプレースホルダ
- HealthKit / Widget / Live Activity / AI / 食事推定は基盤のみ・実装なし
- `simctl install/launch` 経由のスモークテストは環境によって不安定（`xcodebuild test` は問題なし）
