# 手動 QA チェックリスト（v2 / `main`）

Xcode で Scheme `Pulse Cue` を選択して以下を順に確認します。所要時間 10 分程度。

## 起動 / 全体

- [ ] ⌘R で起動 → 「今日 / ワークアウト / ランナー / 履歴 / 設定」の 5 タブが表示される
- [ ] 起動直後にホーム画面に戻る → AppIcon に PulseCue のパルス線アイコンが表示される（Dark / Tinted モードでも崩れない）
- [ ] アプリを kill → 再起動で SwiftData の状態が保持される（DayLog、ルーティン、履歴）

## Today タブ

- [ ] 6 カード（ワークアウト / 栄養 / 運動消費 / 睡眠 / 体重）+ バランス + 「週間サマリー →」リンク
- [ ] 未入力カードはオレンジ枠でハイライト、ボタンが「入力」表示
- [ ] 任意のカードを起動直後に即タップ → 空シートが出ず、入力フォームが表示される

- [ ] 摂取 / 消費を入力 → バランスカードが `intake − exercise` で更新される
- [ ] 「週間サマリー →」をタップ → `HealthSummaryView` の各セクションが表示される（データ不足時は `—`）

## Today: ジムからメニュー作成カード（P4）

- [ ] Today タブに「ジムからメニュー作成」カードが表示されている（ワークアウト開始ボタンと指標カード群の間）
- [ ] ジム未登録時: 「ジムを登録すると、設備に合わせたメニューを作成できます」+「ジムを登録する」CTA → タップで既存のジム検索/登録シートが開く

- [ ] アクティブなジム + マシン保存済み: ジム名 + 「N 台のマシンを使ってメニューを組みます」+「部位を選んで作成」→ タップで `TargetBodyPartSelectionView` に直接遷移
- [ ] アクティブなジムだがマシン未登録: 「このジムのマシン情報がまだありません」+「マシン情報を準備する」→ `MyGymHomeView` 経由でマシン選択にアクセスできる
- [ ] 公式URLが登録されているジムでマシン未登録のとき、補足文「公式URLからの自動取り込みは今後対応予定です。」が表示される
- [ ] 部位選択 → `GeneratedPlanPreviewView` → 「ルーティンとして保存」のフローが従来どおり動く
- [ ] Today タブの他カード（ワークアウト / 栄養 / 運動消費 / 睡眠 / 体重 / バランス / 週間サマリー）は変わらず動く
- [ ] Settings → マイジムからの既存フローは引き続き動く

## Nutrition: 食事カロリーの整合性（P0 reliability）

> Today の摂取値 / バランス と Nutrition タブの合計が**常に一致する**ことが要件。
> 確定 (`.confirmed`) のみが合計に加算される。
> 確認待ち (`.pending`, AI 推定) は合計に影響しない。

- [ ] Nutrition タブで食事を追加 (例 朝食 400 kcal、昼食 800 kcal) → 合計 1200 kcal が即時反映される
- [ ] Today タブに戻ると 摂取 1200 kcal / バランス が同じ値で更新されている
- [ ] 1 件をスワイプで削除 → Nutrition の合計が新しい値 (800 kcal) になる + Today も同じ値に
- [ ] **最後の確定済み食事**を削除 → Today の摂取が空 (— 表示) になり、古い 800 kcal が残らない
- [ ] 食事を編集して kcal を変更 → Nutrition と Today の合計が新しい値に同時に追随する
- [ ] AI 推定 (確認待ち) を追加しても摂取合計は変わらない (確定するまでカロリーには加算されない)
- [ ] AI 推定を破棄しても確定済みの合計は変わらない
- [ ] AI 推定を確定 → 確定済みカロリーに正しく加算される
- [ ] アプリを再起動 → 同じ日の合計が保たれる
- [ ] 朝食 / 昼食 / 夕食 / 間食 をまたいで複数件追加 → 合計が正しく加算される
- [ ] 昨日の食事と今日の食事が混ざらない (ローカル日付でグルーピング)
- [ ] 一連の操作中、外部通信は発生しない（AI ネットワーク呼出は無し）

## Runner 復帰 / P0 reliability

- [ ] ルーティンを開始 → エクササイズ実行 → Complete で休憩に入る
- [ ] 休憩中にアプリをバックグラウンド (ホーム / アプリスイッチャー) へ
- [ ] **休憩終了前** に復帰 → 残り休憩秒が当時の `restDeadline` から再計算され、タイマーが続行する
- [ ] **休憩終了後** に復帰 → 自動的に次のセット / ステップに進み、画面が `exercise` 状態 / `restDeadline` 無しになる
- [ ] (実機で可能なら) アプリを force quit → 再起動 → セッション継続状態が復元される（ルーティン名・現在ステップ / セット index が正しい）
- [ ] 完了済 (`.completed`) セッションをユーザーが意図せず再開させようとしても自動で復元されない（無効化されたセッションは静かに破棄される）
- [ ] 復帰後に通知が二重に飛ばない（同一 sessionId に対する `rest.<sessionId>` 通知は最新だけが残る）
- [ ] 復帰後の Runner UI が「現在ステップ / 現在セット / 残り休憩秒」 を正しく示しており、ユーザーが迷わない
